# 客户端运行生成的文件说明

打包后在客户端运行、由游戏**写入**生成的文件及对应调用关系如下。  
（仅列“会新生成或修改”的文件；只读资源、语言包等不列。）

---

## 一、存储根目录（按平台）

写入路径由各平台 Launcher 在启动时调用 `FileUtils.setDefaultFileProperties(type, basePath)` 设定：

| 平台 | 设定位置 | 典型路径 |
|------|----------|----------|
| **Desktop** | `DesktopLauncher.main()` | Windows: `%APPDATA%\.coladungeon\`（vendor 来自包名）<br>Mac: `~/Library/Application Support/.coladungeon/`<br>Linux: `$XDG_DATA_HOME/.coladungeon/` 或 `~/.local/share/.coladungeon/` |
| **Android** | `AndroidLauncher.onCreate()` | `Files.FileType.Local` → 应用私有存储（如 `/data/data/.../files/`） |
| **iOS** | `IOSLauncher` | `Files.FileType.Local` → 应用沙箱 |

以下文件名均相对于该 **basePath**（Desktop 上即“游戏数据目录”）。

---

## 二、生成的文件列表

| 文件名 | 说明 | 由谁写入（类与方法） | 触发时机简述 |
|--------|------|----------------------|--------------|
| **global.json** | 全局数据：图鉴、成就、排行榜、Journal、Badges 等 | `SaveManager.saveGlobal(Bundle)` | 见下表「调用 saveGlobal 的类」 |
| **save-001.json** … **save-042.json** | 各存档槽游戏数据（含关卡等） | `SaveManager.saveGame(slot, Bundle)` → 内部用 `FileUtils.bundleToFile(filename, bundle)` | `Dungeon.saveAll()` → `Dungeon.saveGame()` / `saveLevel()`；选关、过关、退出等 |
| **save-xxx.json.tmp** | 写入存档时的临时文件 | `FileUtils.bundleToFile()` 内部（先写 .tmp 再 moveTo 正式文件） | 保存过程中短暂存在，正常结束会替换为对应 `save-xxx.json` |
| **bones.dat** | 当前“遗物”数据（英雄死亡时留下的一件物品） | `Bones.leave()` → `FileUtils.bundleToFile(BONES_FILE, bundle)` | 英雄死亡时 `Hero.die()` → `Bones.leave()` |
| **keybinds.dat** | 按键/键位绑定 | `CDKeyBinding.saveBindings()` → `FileUtils.bundleToFile(BINDINGS_FILE, b)` | 玩家在按键设置界面保存时（如 `WndKeyBindings` 中调用 `CDKeyBinding.saveBindings()`） |
| **settings.xml** | 游戏设置（分辨率、音量、语言等，CDSettings 继承 GameSettings） | 通过 `Preferences` API 写入；Desktop 上为 `Lwjgl3Preferences` 指向 `basePath + DEFAULT_PREFS_FILE` | 任何 `CDSettings.xxx(value)` / `GameSettings.put()` 的调用都会 flush 到该文件 |

**说明：**

- `global.json`、`save-xxx.json`、`bones.dat`、`keybinds.dat` 均通过 **`FileUtils.bundleToFile(fileName, bundle)`** 落盘，底层是 `com.watabou.utils.FileUtils`（SPD-classes）。
- `settings.xml` 由 LibGDX `Gdx.app.getPreferences(DEFAULT_PREFS_FILE)` 提供；Desktop 在 `DesktopLauncher` 里把 `CDSettings` 绑到 `basePath` 下的 `settings.xml`，因此也落在同一游戏数据目录。

---

## 三、谁在什么时候写这些文件

### 1. SaveManager.saveGame(slot, bundle) → save-001.json 等

- **调用方**：`Dungeon.saveGame(save)`（在 `Dungeon.saveAll()` 流程里）。
- **触发场景**：选关、过关、退出游戏等会调用 `Dungeon.saveAll()` / `saveLevel()` 时。

### 2. SaveManager.saveGlobal(bundle) → global.json

- **调用方**（多处）：
  - `WelcomeScene`（进入游戏/初始化时写全局）
  - `Rankings`（排行榜变更）
  - `Journal`（日志/图鉴）
  - `Badges`（成就）
  - `Assets`（资源相关全局状态）
  - `WndLuaScript`（Lua 脚本相关全局数据）

### 3. Bones.leave() → bones.dat

- **调用方**：`Hero.die()` 内部调用 `Bones.leave()`。
- **触发**：英雄死亡时写入当前“遗物”数据。

### 4. CDKeyBinding.saveBindings() → keybinds.dat

- **调用方**：`WndKeyBindings` 等按键设置界面在用户确认保存时调用 `CDKeyBinding.saveBindings()`。

### 5. settings.xml（Preferences）

- **写入**：任何对 `CDSettings.*` / `GameSettings.put*` 的修改（如分辨率、音量、语言、键位开关等），都会通过 `get().flush()` 写回。
- **初始化**：Desktop 在 `DesktopLauncher` 中通过 `CDSettings.set(Lwjgl3Preferences(...))` 指定文件路径为 `basePath + "settings.xml"`。

---

## 四、类与文件对应关系简表

| 生成的文件 | 直接写入的类 | 使用的方法/API |
|------------|--------------|----------------|
| global.json | SaveManager | saveGlobal() → FileUtils.bundleToFile(GLOBAL_FILE, bundle) |
| save-001.json … save-042.json | SaveManager | saveGame() → FileUtils.bundleToFile(filename, bundle) |
| save-xxx.json.tmp | FileUtils（内部） | bundleToFile() 中临时文件 |
| bones.dat | Bones | leave() → FileUtils.bundleToFile(BONES_FILE, bundle) |
| keybinds.dat | CDKeyBinding | saveBindings() → FileUtils.bundleToFile(BINDINGS_FILE, b) |
| settings.xml | GameSettings / CDSettings | Preferences.put* + flush（Desktop 为 Lwjgl3Preferences 写 basePath + DEFAULT_PREFS_FILE） |

如需排查“某个文件是谁写的、什么时候写”，可按上表找到对应类，再在工程里搜索该类名与上述方法名即可定位调用链。
